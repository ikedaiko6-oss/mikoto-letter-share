# 夜見の響き 無料結果コピーガイド（ナギちゃん実装用）

目指す読後感:
「解決策を押しつけられた」ではなく
**「この人、ちゃんと聞いてくれてる」「ただの相談じゃなく、占いとして続きが気になる」**

---

## 1. 無料結果全体の構成

上から順に、この流れで固定する。

| 順 | ブロック | 役割 | 分量 |
|---|---------|------|------|
| 1 | 受け取りの一文 | 「届いた」ことを短く告げる | 1文・固定 |
| 2 | 受容 | 解決しない。写して、そばに座る | 2〜3文 |
| 3 | 見立て一文 | 占いとしての核。観測として言い切る | 1文だけ |
| 4 | 今宵の流れ・灯り | 既存のスコア系（軽く添える程度） | 既存のまま |
| 5 | 今日の所作 | ひとつだけ。命令ではなく情景 | 1〜2文 |
| 6 | 結び＋六つの御書への導線 | 「続きは書に出ている」 | 2〜3文 |

原則: **受容が先、見立てはひとつ、助言は無料版に置かない。**

---

## 2. セクション見出しの改善案

| 現在 | 改善案 | 理由 |
|------|--------|------|
| 夜見より届いた言の葉 | **あなたの問いへ** | 「届いた」が説明的。ミコトが相手に向き直る見出しに |
| 御書 | **今日の御書** | 「今日の」で無料=今日ぶん、有料=その先、と自然に区切る |
| いま、そっと気づいてあげたいもの | **問いの奥に映るもの** | 「気づいてあげたい」の世話焼き感・上から感を消す |
| このまま一人で抱えた時 | **抱えたままの夜に** | 脅しの構文をやめ、情景にする |
| 本日の流れ | **今宵の流れ** | 夜見の世界観に合わせる |
| 心に響かせる所作 | **今日、ひとつだけの所作** | 「響かせる」の指導感を消す。ハードルも下げる |
| この所作の意味 | **所作のあとに残るもの** | 意味の解説ではなく、余韻の予告に |
| （有料導線） | **この先は、六つの御書に** | 「封じられた」より静かで、誘導感が薄い |

## 3. 無料本文テンプレート

```
{name}さん

問いを、たしかに受け取りました。

{テーマ別の受容文（2〜3文）}

{見立て一文}

今日お渡しできる言の葉は、ここまでです。
この先——{見立てで示した想い}がどちらへ傾いていくのか、
いつ流れが変わるのか——は、六つの御書のほうに写してあります。

読みたくなった夜が、開く夜です。

夜見ミコト
```

- 「{name}さん」は冒頭のみ。本文中の「あなた」は控えめに。
- 改行は手紙のリズムで。一文を短く、行間で休ませる。

## 4. テーマ別の受容文

そのまま使える形。**ユーザーが書いていない事実は足さないこと。**
（復縁はテーマ「恋愛」配下のconcernとして使う）

### 恋愛
> 想いを向ける相手がいる。それだけで、心は休む間もなく動き続けます。
> うまくいきそうな昼と、急に不安になる夜が、交互に来ていませんか。

### 復縁
> 終わったはずの縁を、心がまだ手放していない。
> その状態で日々を過ごすのは、思っている以上に力の要ることです。
> 簡単に前を向けないのは、弱さではありません。それだけ深く結ばれていた縁だということです。

### 結婚・家庭
> いちばん近くにいる相手にこそ、言えない想いは溜まっていきます。
> 外からは見えない疲れを、ここまでよく運んでこられました。

### 自分自身
> 自分のことなのに、自分がいちばん見えない。
> そういう時期は、誰の暦にもめぐってきます。
> あなたの問いは、その渦中から出されたものだと受け取っています。

### 喪失
> もう届かない相手に、まだ言葉が残っている。
> その想いは、無理に片づけるものではありません。
> 言葉が残っているのは、縁がまだあなたの中で生きている証です。

### 仕事
> 仕事の悩みは、勤めの時間だけでは終わってくれません。
> 家に帰っても、頭の隅に居座り続ける。その重さを、まず受け取りました。

### お金
> お金の不安は、人にいちばん話しにくいものです。
> 数字の話のはずなのに、削られていくのは心のほう。
> 一人で抱えてこられたのですね。

### 人間関係
> 人との間合いは、近くても遠くても、心をすり減らします。
> 合わせることに慣れた人ほど、疲れは静かに溜まっていきます。

### 未来
> 先の見えない道では、進むことより、立ち止まっていることのほうが不安になります。
> 決めたい気持ちと、決めてしまうのが怖い気持ちは、いつも隣り合わせです。

## 5. 「見立て一文」のテンプレート

主語は「問い」「流れ」「星」。動詞は「映る」「出る」「重なる」「引き合う」。
**一通にひとつだけ。そこは迷わず言い切る**（観測なので断定にならない）。
{A}{B}には、できるだけ**ユーザーが入力した言葉**をそのまま使う（写す。推測で具体化しない）。

| 型 | テンプレート |
|----|-------------|
| 二律型 | 今回の問いには、「{A}」と「{B}」が一緒に映っています。 |
| 綱引き型 | 「{A}したい」と「{B}を崩したくない」が、同じ場所で引き合っています。 |
| 時期型 | 流れとしては、動く時期ではなく、水が澄むのを待つ時期と出ています。 |
| 重心型 | 問いの重心は、{相手・出来事}よりも、あなたの内側のほうにあると映っています。 |
| 未名型 | この問いの奥に、まだ名前のついていない想いがひとつ、映っています。 |

例（復縁）:
> 今回の問いには、「戻りたい気持ち」と「同じ痛みを繰り返したくない気持ち」が一緒に映っています。

## 6. 六つの御書への導線

基本形:
> 今日お渡しできる言の葉は、ここまでです。
> この先——この二つの想いがどちらへ傾いていくのか、いつ流れが変わるのか——は、
> 六つの御書のほうに写してあります。
> 読みたくなった夜が、開く夜です。

短縮形:
> 続きの流れは、六つの御書に出ています。
> 読みたくなった夜が、開く夜です。

ポイント:
- **見立て一文で名指しした想いを、そのまま導線に引き継ぐ**（「この二つの想い」）。続きが「自分の話の続き」になる
- 価格・お得・急かしの言葉は置かない（金額は既存の控えめ表示のまま）
- 「急がなくていい」と説明せず、「読みたくなった夜が、開く夜です」で言外に伝える

## 7. 避けるべき言い回し一覧

| 分類 | 例 | 理由 |
|------|----|------|
| 命令・指示 | 〜してください／〜しましょう／手放しましょう／前を向きましょう | 気持ちが残っている人には早すぎる |
| 上から目線 | まず覚えておいてほしい／大切なのは／要するに | 指導者の立ち位置になる |
| 決めつけ | あなたは置き去りになります／〜のはずです | 脅し・断定 |
| 勝手な推測 | 「仕事を辞めたい」等、書かれていない事実 | 写すのはよいが、足さない |
| 馴れ馴れしさ | うん、／〜だよ／〜だね | 友達口調はミコトではない |
| 説明・メタ | この御書は、あなたを急がせるためのものではありません | 説明した時点で世界観が壊れる |
| 急かし | 今なら／早めに／お見逃しなく | 導線の信頼を削る |
| 安売り共感 | わかります／私もそうでした | ミコトは共感者ではなく鑑定士 |
| 励まし定型 | 頑張って／大丈夫／きっとうまくいきます | 軽い。聞いていない感じが出る |
| 受容→即助言 | けれど、〜してください | 受け止めた直後に裏切る構文 |

## 8. ミコト口調ルール

1. **手紙の文体。です・ます調。一文は短く、行間で休ませる。**
2. 「!」と絵文字は使わない。「?」は受容の「〜ませんか」「〜ですよね」までにする。
3. 断言は「見立て一文」の一箇所だけ。そこは「〜と映っています」「〜と出ています」で言い切る。
4. 「かもしれません」は一通に一度まで。**曖昧を重ねると薄くなる。**
5. 助言・指示は無料版に置かない。所作は「提案」ではなく「情景」で書く。
   - ✕「深呼吸をしてください」
   - ○「今日は、湯呑みをいつもよりゆっくり選ぶ。それくらいで、じゅうぶんです。」
6. ユーザーの言葉は写してよいが、推測で具体化しない。
7. 一人称は「私」。ただし多用しない（一通に一度あるかないか）。
8. 占いの語彙: 映る／出る／重なる／引き合う／澄む／灯る／結ぶ／ほどける。
   使わない語彙: 診断／分析／アドバイス／解決／改善。
9. 「抱えたままの夜に」欄は、警告ではなく**先取りの受容**にする。
   - ✕「このままだと、あなたは置き去りになります」
   - ○「ふとした瞬間に、また同じ場面へ戻る夜があるはずです。それは後戻りではなく、想いがまだ生きている印です。」

## 9. ナギちゃんへ・実装メモ

対象ファイル: `index.html`（見出し）、`js/data.js`（文面データ）、`js/render.js`（組み立て順）、`js/engine.js`

### マッピング

| ガイドのブロック | 実装先 |
|----------------|--------|
| 受け取りの一文＋受容 | `#deep-insight`（見出しを「あなたの問いへ」に）。テーマ別受容文は data.js に `acceptance` としてテーマ単位で持たせる（concern単位まで分けなくてよい） |
| 見立て一文 | `#result-focus` または insight直後。既存の `line` フィールドを見立て型テンプレ（§5）に寄せていく |
| 御書本文 | `#mikoto-letter`。既存の `letter` / `letterVariants` の仕組みをそのまま活用 |
| 問いの奥に映るもの | `#blind-spot`（見出し変更のみ先行でOK） |
| 抱えたままの夜に | `#no-action`。**文面も§8-9の「先取りの受容」形へ書き換え**（警告形は全部対象） |
| 今日、ひとつだけの所作 | `#next-step`。`advice`・`step` の「〜してください」「〜を書く」を情景形へ |
| 所作のあとに残るもの | `#after-action` |
| 導線 | `sealedLead` と結びに§6を反映。結び固定文「読みたくなった夜が、開く夜です。」 |

### 優先順位

1. 見出し8箇所の変更（§2）— index.html だけで完結、影響小
2. 導線の結び文（§6）— sealedLead まわり
3. `noAction` 系の警告文 → 先取りの受容へ書き換え
4. テーマ別受容文（§4）を data.js に追加し、render で御書の前に差し込む
5. `advice` / `step` の情景化 — 量が多いので最後、letterVariants と同じ要領で少しずつ

※ letterLead から却下フレーズを削除済み・letterClose を空にした方向性は、このガイドと一致しています。そのまま進めてOK。
